# Push-уведомления через Telegram-бота — инструкция по установке

Это серверная часть (Firebase Cloud Functions), которая следит за базой данных
и шлёт сообщения через вашего бота, даже когда приложение закрыто.
Токен бота НЕ хранится в HTML — он лежит отдельно в Firebase Secret Manager.

Что уже сделано в `app.html` (клиент):
- При входе через Telegram сохраняется `telegramId` пользователя в `users/{id}`.
- В профиле появилась кнопка «🔔 Уведомления в Telegram: вкл/выкл» — переключает
  флаг `notificationsEnabled` в базе. Функции ниже проверяют этот флаг перед отправкой.

## Шаг 1. Установить Firebase CLI (один раз)

```bash
npm install -g firebase-tools
firebase login
```

## Шаг 2. Разместить файлы

Положите рядом эти файлы/папки (они уже подготовлены):
```
firebase.json
.firebaserc            # уже указывает на проект book-2b50d
functions/
  index.js
  package.json
```

## Шаг 3. Установить зависимости функций

```bash
cd functions
npm install
cd ..
```

## Шаг 4. Перевести проект на план Blaze (pay-as-you-go)

Cloud Functions 2-го поколения (которые дергают внешний Telegram API) требуют
план Blaze. Он по-прежнему бесплатный при небольшой нагрузке — просто нужна
привязанная карта. Сделать это можно в консоли Firebase:
Project settings → Usage and billing → Modify plan.

## Шаг 5. Сохранить токен бота как секрет (НЕ в коде!)

```bash
firebase functions:secrets:set TELEGRAM_BOT_TOKEN
```
Введите токен от @BotFather, когда попросит. Он будет зашифрован и доступен
только вашим функциям.

## Шаг 6. Деплой

```bash
firebase deploy --only functions
```

После деплоя в консоли Firebase (Functions) появятся три функции:
- `notifyNewPost` — новый пост в ленте → всем, кроме автора
- `notifyNewMessage` — новое сообщение в чате/группе → участникам, кроме отправителя
- `notifyNewComment` — новый комментарий → автору поста

## Важный нюанс

Telegram позволяет боту писать пользователю, только если тот **хотя бы раз
открывал чат с ботом** (нажимал Start или открывал Mini App через него).
Если у пользователя не установлен `telegramId` (например, он вошёл по
логину/паролю, а не через Telegram) — уведомления ему просто не шлются,
это ожидаемо и безопасно (ошибка логируется, но не ломает остальное).

## Проверка

1. Откройте приложение через Telegram, зайдите в Профиль — должна появиться
   кнопка с уведомлениями (для веб-логина её не будет — это нормально).
2. Закройте приложение и попросите кого-то (или сами со второго аккаунта)
   опубликовать пост / написать в чат / оставить комментарий.
3. Вы должны получить сообщение от бота в Telegram в течение нескольких секунд.
4. Логи в случае проблем: `firebase functions:log`.
